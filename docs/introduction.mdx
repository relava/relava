# What is Relava?

Relava is a **local package manager and registry** for Claude Code resources. It manages the prompt-layer artifacts that shape how Claude thinks and behaves: skills, agents, commands, and rules.

Think `npm` or `brew`, but for Claude Code extensions — and everything runs locally on your machine.

## The Problem

Claude Code's extension model is file-based. Skills are directories, agents are `.md` files, commands are `.md` files, rules are `.md` files. There is no built-in package manager, no versioning, no dependency tracking, no discovery mechanism.

Developers manually copy files between projects. There's no way to:

- **Share** a useful skill with your team
- **Version** your agents so you can roll back a bad change
- **Declare** what resources a project needs in a single file
- **Update** resources across projects when improvements land
- **Discover** what resources exist

## The Solution

Relava solves this by treating Claude Code resources as first-class packages:

## Key Design Principles

1. **Local-first.** Everything works offline. No account required. No cloud dependency.
2. **Prompt-layer only.** Relava manages text and files that get injected into Claude's context. It does not manage infrastructure (MCP servers, runtimes, databases).
3. **Non-invasive.** Relava writes files to standard Claude Code locations. If you uninstall Relava, your resources keep working.
4. **Multi-file aware.** Skills can contain templates, libraries, and support files. Relava handles the full complexity.
5. **Enterprise-ready.** REST-first architecture with storage abstraction traits. Switch from local to enterprise registry by changing one URL.

## How It Works

```
CLI (relava) ──REST──> Registry Server (localhost:7420)
     │                       │
     │                  Resource Store (~/.relava/store/)
     │                  SQLite Metadata DB
     │                  Web GUI
     v
Project Filesystem (.claude/)
```

The **CLI** reads your project's `relava.toml`, fetches resources from the **registry server**, and writes files to your project's `.claude/` directory. The **web GUI** lets you browse and search the registry visually.
